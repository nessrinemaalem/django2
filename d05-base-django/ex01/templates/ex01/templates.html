Ex01 : Moteur de template.

1. Les blocs

Les blocs définissent des zones personnalisables dans un template de base. Exemple : 

{% verbatim %}{% block content %}
Contenu ici
{% endblock %}{% endverbatim %}

Dans un template enfant, le contenu de ce bloc remplace celui du template de base.

Exemple de boucle

{% verbatim %}
{% for item in items %}
  
{{ item }}

{% endfor %}
{% endverbatim %}
    {% for item in items %}
  • {{ item }}
    • {% verbatim %}{% empty %}
  • Aucun élément à afficher
    • {% endverbatim %} {% endfor %}

3. Conditions if

Pour afficher du contenu selon une condition :

{% verbatim %}{% if user.is_authenticated %}
  Bonjour, {{ user.username }}
{% else %}
  Bonjour, invité
{% endif %}{% endverbatim %}

4. Affichage de variables

Les variables passées depuis la vue sont accessibles dans le template :

{% verbatim %}<p>Nom : {{ name }}</p>{% endverbatim %}

Résumé

Le moteur de templates Django permet donc de :

  • Réutiliser des structures HTML grâce aux blocs et à l’héritage de templates.
  • Afficher dynamiquement des données avec les variables du contexte.
  • Ajouter de la logique simple avec les boucles et conditions.
