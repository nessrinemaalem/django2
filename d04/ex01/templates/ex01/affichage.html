Ex01 : Processus d’affichage d’une page statique.

Voici le processus simplifié :

  1. Requête HTTP : Un utilisateur navigue vers une URL, par exemple /ex01/process/.
  2. Routing : Django regarde dans urls.py de l’application et trouve la vue associée à cette URL.
  3. Vue : La fonction dans views.py s’exécute. Elle peut préparer des données et retourne un HttpResponse ou render(request, template, context).
  4. Template : Le fichier HTML (ici notre `base.html` étendu par `affichage.html`) est chargé et les blocs sont remplacés par le contenu de la page.
  5. Réponse : Le HTML final est envoyé au navigateur, qui l’affiche à l’utilisateur.

En résumé, l’utilisateur demande une page → Django dirige la requête vers la vue → la vue utilise un template → le HTML rendu est renvoyé au navigateur.
